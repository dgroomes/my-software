# software-decision-records

This directory contains my *Software Decision Records* (SDRs).


## Overview

SDR is a made up term. I don't want to over-index on ADRs (Architecture Decision Records) and such, but in the day and age of agentic implementation, I need to offset the fast and chaotic sprawl of code with the slow and clear direction of a decision record.

I've never been fond of the A in ADR because it's like, why does it have to be specific to architecture? The name of a project, for example, is an important decision that might be the product of lots of ideas and thought. That's literally perfect to encode the thought process in a decision record, but it has nothing to do with architecture.

On the other hand, I don't want to call it just "decision records" because I don't want to give it too much broad power. It stays in a lane. The lane is this `sdr/` directory and the sophistication is only as high as markdown. No big rules.
